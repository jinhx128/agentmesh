## Blocking (real regressions)

**[Blocking] `packages/cli/src/commands/studio.ts:39–40` — default `assetDir` does not match built layout or docs**

`resolveDefaultStudioAssetDir` uses  
`new URL("../../../../apps/studio-web/frontend/", import.meta.url)`.

For the emitted CLI at `dist-node/packages/cli/src/cli.js`, four `..` segments resolve to the **package root** (parent of `dist-node/`), then `apps/studio-web/frontend` → **`<repo>/apps/studio-web/frontend`**, not **`<repo>/dist-node/apps/studio-web/frontend`**.

That contradicts `README.md` (“`dist-node/apps/studio-web/frontend`”), `plan.md`, sidecar expectations, and the pack manifest (`dist-node/apps/studio-web/frontend/index.html`). It would make `agentmesh studio` fail to find `index.html` after a normal root `npm run build` / `build:studio-frontend`, unless `AGENTMESH_STUDIO_ASSET_DIR` is set.

**Expected shape (for this layout):** from `dist-node/packages/cli/src/`, use **three** `..` segments to reach `dist-node/`, then `apps/studio-web/frontend` (i.e. `../../../apps/studio-web/frontend/`), matching `@agentmesh/studio` / `studio-desktop` resolution style.

Until this is fixed, treat **`agentmesh studio` default asset serving as broken** in both dev and packed installs.

## Should fix (non-blocking / hardening)

**[Should Fix] `apps/studio-web/vite.config.ts:8` — `outDir` tied to `process.cwd()`**

`path.resolve(process.cwd(), "dist-node", "apps", "studio-web", "frontend")` is correct when Vite is invoked from the **repo root** (`npm run build:studio-frontend`), but if someone runs `npm run build` **inside** `apps/studio-web`, output lands under `apps/studio-web/dist-node/...`, which is easy to mistake for the canonical tree and confusing next to root `dist-node/`.

Prefer resolving `outDir` from `import.meta.url` (monorepo root or fixed `../../dist-node/...`) so the path is stable regardless of cwd.

**[Should Fix] `scripts/check-boundaries.mjs:18` — `vite.config.ts` not under `apps/studio-web/src/`**

The stricter Studio frontend rules only apply to `apps/studio-web/src/`. `apps/studio-web/vite.config.ts` is still tooling at repo boundary; today it’s probably fine, but if you want “frontend import boundaries” to cover the Vite entry, extend the guard to `apps/studio-web/` (with an explicit carve-out for allowed `node:` imports in config).

**[Should Fix] Doc drift (historical paths)**

`plan.md` still mentions `dist-node/apps/studio/frontend` in older bullets; worth aligning with `dist-node/apps/studio-web/frontend` so future readers don’t follow the wrong path. (Non-blocking to runtime.)
