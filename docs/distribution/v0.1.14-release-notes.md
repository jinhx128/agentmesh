# AgentMesh v0.1.14

## 新功能与体验

- 增加跨 Codex、Cursor、Claude Code、Antigravity 和 OpenCode 入口的 Reviewer Scope 协议、本机 session registry、lease、TTL、关闭与过期清理命令，以及 Studio 安全管理入口。
- 增加 reviewer session provenance、non-hermetic 传播、bounded fresh fallback 和 Release Check 独立性门禁；packet schema 与 workflow recipe 继续保持版本 1 兼容。
- Studio 统一 Run/Call 活动导航，分组默认预览五项并支持展开，补充运行/调用标签、当前节点状态、单项删除和分组复位体验。
- Settings / Integrations 可以自动检测 PATH-visible CLI，并直接安装或更新公共 npm CLI，无需用户输入路径；Desktop 增加自动检查应用更新偏好和原生更新入口。
- 全面同步 AgentMesh Skill、README、落地页和 Studio Manual 的跨宿主调用、安全边界与管理命令。

## Reviewer Session 默认门禁

- P5 真实 A/B 没有产生合格的 resumed arm，provider `tool_reads` 也不可观测，因此 Codex、Cursor、Claude Code、Antigravity 和 OpenCode 五个内置 reviewer provider 均保持 fresh-only。
- Claude Code/OpenCode 仅保留底层结构化 probe，并只允许 disposable fake-CLI 测试通过实验环境变量启用；生产默认不会读写 reviewer session registry。
- 正式 review、release、安全、合规和审批 gate 必须使用 `independent`，原始 host/provider session ID 不进入 packet、日志、Studio、git 或聊天。

## 兼容与升级

- packet schema、workflow recipe 和现有 CLI 安装命令保持兼容；新增字段均为 optional，旧 packet 继续可读。
- CLI 与 Desktop 是独立渠道：CLI 可从 npm 或同版本 GitHub tarball 更新；Desktop 可从 DMG 完整替换，也可由已安装的 updater-enabled 版本使用应用内更新。
- macOS DMG 面向 Apple Silicon，未做 Apple Developer ID 签名或 notarization；Tauri updater archive 使用独立更新密钥签名。
