# Skill Output Contract

`schema_version`: 1

AgentMesh Skill output is the host-facing natural-language contract generated by
`agentmesh skill show`, `agentmesh skill export --format markdown`, and
`agentmesh skill install --target <host>`.

## Required Metadata

Canonical Skill markdown must begin with YAML frontmatter containing the skill
`name` and `description`, so hosts that validate `SKILL.md` files can load it.
Host wrapper files, such as Cursor rules or Antigravity extension files, may add
their own outer format around the canonical Skill markdown.

Generated Skill markdown must include a `Version Metadata` block immediately
after the top-level `# AgentMesh Skill` heading.

Required fields:

- `AgentMesh CLI version`: package version of the installed CLI.
- `Packet schema version`: supported packet/status/event/artifact schema version.
- `Workflow recipe schema version`: supported workflow recipe schema version.

The metadata block is generated by `@agentmesh/skills` through
`agentmeshSkillMarkdown`, not hand-maintained in each host install wrapper.
Host-specific wrappers must embed the same canonical markdown so exported and
installed Skill content agree.

## Compatibility Policy

- Entry agents should compare the metadata block before relying on older copied
  Skill text.
- A newer packet or workflow recipe schema version than the local CLI supports
  is not compatible.
- A different CLI patch version can still be used when the packet and workflow
  recipe schema versions match.
- Unknown or missing metadata means the Skill should be refreshed with
  `agentmesh skill install --target <host> --force` or regenerated with
  `agentmesh skill export --format markdown`.

## Observation Surface

- `agentmesh skill show` writes canonical markdown to stdout.
- `agentmesh skill export --format markdown` writes the same markdown to stdout.
- `agentmesh skill install --target <host>` writes the expected project-level
  host file and verifies its content against the canonical markdown. Codex,
  Cursor, Antigravity CLI, and OpenCode use
  `.agents/skills/agentmesh/SKILL.md`; Claude Code uses
  `.claude/skills/agentmesh/SKILL.md`.
- `agentmesh skill verify --target <host> --json` reports installed file status
  but does not mutate files. Legacy `.cursor/rules/agentmesh.mdc` is reported
  as legacy-only evidence when present and is not deleted by install.
- `@agentmesh/skills` exposes `expectedSkillFilesForTarget` for read-only
  diagnostics that need the same project-level target paths without writing
  host files.
