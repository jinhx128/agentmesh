# Studio 统一活动导航与展示标题设计

## 目标

Studio 左侧导航不再把运行和调用拆成两个入口，而是把它们作为同一种“活动”按时间混排。每条活动使用可读的展示标题，技术 ID 继续承担目录名、CLI 定位和关联键职责。

本设计同时满足以下约束：

- 用户可以显式指定标题，也可以完全不关心标题。
- 主控 Agent 在用户未指定标题时，默认生成简洁中文标题。
- AgentMesh 不为标题额外调用模型，不增加鉴权、地区、时延和费用风险。
- 即使最终没有收到 Agent 生成的标题，运行和调用仍能正常创建。
- 旧记录不迁移、不改写，仍可被新版 Studio 读取。

## 标题模型

### 技术 ID 与展示标题分离

运行继续使用 `run_id`，调用继续使用 `call_id`。现有磁盘目录、CLI 参数、URL、关联字段和选择键均不改用标题。

新建记录持久化独立的 `title` 字段：

- 运行标题写入运行状态数据，并由运行列表 API 返回。
- 调用标题写入调用记录，并由调用列表 API 返回。
- 读取历史记录时允许 `title` 缺失；Studio API 对外提供可展示值。

标题不是唯一键，允许重名。

### 标题优先级

创建运行或调用时按以下优先级解析标题：

1. 用户显式提供的标题。
2. 主控 Agent 根据任务或调用目的生成并通过 `--title` 传入的中文标题。
3. AgentMesh 运行时生成的确定性默认标题。

主控 Agent 的调用规范要求：如果用户未给标题，根据请求生成 4 至 24 个字符的中文摘要，再调用 AgentMesh。该步骤由已经在处理用户请求的主控 Agent 完成，不产生额外模型调用。

CLI 的 `--title` 保持可选。缺少、空白或清理后为空时不报错，进入运行时默认标题逻辑。

### 默认标题

运行时默认标题使用 ASCII 短横线 `-` 连接工作区目录名和摘要，两侧不加空格：

- 运行：`工作区名-任务首句摘要`
- 调用：`工作区名-调用目的摘要`
- 无可用摘要：`工作区名-HH:mm:ss`

示例：

- `agentmesh-优化活动列表`
- `ai-config-审查模型配置`
- `agentmesh-17:26:08`

默认摘要只做本地确定性处理：选择第一个有效文本来源、提取首个非空内容、去掉常见 Markdown 前缀、合并空白并限长。它不翻译、不冒充模型总结。

运行的摘要来源依次为任务文本和时间；调用的摘要来源依次为 `purpose`、提示词首个有效内容和时间。工作区目录名无法取得时使用 `workspace`。

主控 Agent 生成标题遵守 4 至 24 个字符的产品约束。CLI 对显式标题只做安全规范化并允许 1 至 80 个 Unicode 字符，避免展示元数据阻断核心任务；Studio 在视觉层截断超长标题，完整值保留在 `title` 属性或详情数据中。

## 统一活动数据模型

Studio 前端把运行摘要和调用摘要投影为统一活动项：

```ts
type StudioActivityItem =
  | { kind: "run"; key: string; title: string; timestamp: string; run: StudioRunSummary }
  | { kind: "call"; key: string; title: string; timestamp: string; call: StudioCallSummary };
```

统一项仅负责左侧导航的排序、分组、搜索和选择路由，不合并右侧详情模型。

时间规则保持可解释：

- 运行使用 `updated_at ?? latest_event_timestamp ?? created_at`。
- 调用使用 `created_at`。
- 有效时间按降序排列；时间相同时使用稳定的 `kind + key` 排序。

## 左侧导航

### 结构

移除顶部“运行 / 调用”分段切换，左侧只保留：

1. 统一搜索框。
2. 统一刷新按钮。
3. 自动刷新间隔。
4. 按日期分组的活动列表。

日期组数量为运行与调用的合计。每组默认展示最新 5 条，底部提供“展开其余 N 条”；展开后显示全部并可“收起到 5 条”。日期组自身原有的整体折叠交互继续保留，两层展开状态互不影响。

搜索作用于标题、技术 ID、工作区以及各类型现有的可搜索字段。搜索期间显示全部匹配项，不应用 5 条预览限制，也不显示组内“展开其余”按钮。

### 活动项

活动项保持两行：

- 第一行：展示标题，单行截断。
- 第二行左侧：仅显示低干扰的 `[运行]` 或 `[调用]` 标签，不再显示摘要或元数据。
- 第二行右侧：显示本地时间。

活动项沿用已确认的轻量文件树视觉：默认无边框，悬停使用轻微灰底，选中使用柔和实色灰底，不使用青色粗边框或左侧强调条。

### 选择路由

点击运行项时：

- 更新选中运行。
- 右侧切换到现有运行详情工作区。
- 保留运行详情页签和制品抽屉行为。

点击调用项时：

- 更新选中调用。
- 右侧切换到现有调用详情工作区。
- 保留调用采纳等既有能力。

设置和手册仍可覆盖右侧工作区；返回选择活动时恢复对应详情。

## 加载、刷新与错误

运行和调用继续使用现有独立 API，但由统一导航同时消费：

- 首次加载并行请求两类数据。
- 手动刷新和自动刷新同时刷新两类数据。
- 一类加载失败时，另一类已成功的数据仍可浏览。
- 局部失败以导航内非阻塞提示呈现；只有两类都失败且无可用数据时显示整体失败状态。
- 任一类型仍在加载且另一类型已有数据时保留已有列表，不用全屏 loading 替换。

刷新不得因为统一列表重新自动切换用户当前选中的活动；只有当前选择已经不存在时，才按既有选择回退规则处理。

## 兼容性

- 旧运行状态没有 `title` 时回退显示 `run_id`。
- 旧调用记录没有 `title` 时回退显示 `call_id`。
- 不批量回写历史记录，避免修改用户运行证据。
- SDK 和服务端读取类型允许旧数据缺少标题；所有新写入路径都保存解析后的标题。
- 现有 `--run-id`、调用 ID、目录布局和关联字段保持不变。

## Agent 调用规范

AgentMesh skill 和共享调用说明增加统一规则：

- 用户给出标题时使用用户标题。
- 用户未给标题时，主控 Agent 先根据任务生成 4 至 24 个字符的中文摘要。
- 主控 Agent 调用 `run`、`flow run` 或 `call` 时通过 `--title` 传入结果。
- 标题只描述目标，不加入模型名、Agent 名、时间戳或状态词。
- 无法可靠总结时可以不传，让 AgentMesh 使用确定性默认标题；不得阻断主任务。

## 测试

### 运行时与 CLI

- 显式标题被规范化并持久化。
- 主控 Agent 传入的中文标题通过 CLI 到达运行和调用记录。
- 未传、空白或无效标题时生成 `工作区名-摘要`。
- 没有有效摘要时生成 `工作区名-HH:mm:ss`。
- 默认标题使用 ASCII `-`，不使用 `·`、`｜` 或长横线。
- 标题不改变技术 ID、目录或关联键。
- 旧记录缺少标题仍能读取。

### Studio

- 顶部不再渲染运行/调用切换。
- 运行和调用按时间稳定混排并按日期合并计数。
- 统一搜索覆盖两类项目。
- 日期组整体折叠与组内 5 条预览保持独立。
- 第二行只有类型标签和时间，不再渲染摘要元数据。
- 选择运行和调用分别进入正确的详情工作区。
- 单类加载失败仍展示另一类数据。

## 非目标

- 不把标题用作 ID 或目录名。
- 不为生成标题增加独立模型请求或配置“命名 Agent”。
- 不自动重命名或回写旧记录。
- 不合并运行详情与调用详情。
- 不在本次变更中修改右侧详情的信息架构。
