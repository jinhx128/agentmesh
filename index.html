三个入口，共用一套本地事实

CLI、Studio 和 Skill 都围绕同一套 packet 文件工作。底层 agent 登录态和模型权限仍由各自工具管理。

CLI

管理 agents、workflows、presets、MCP servers、model aliases 和 run packet。 支持 dispatch、retry、resume、attach、doctor、release-check summary。

agent registry packet runtime doctor

Studio

本地 UI 展示 Runs、Calls、events、artifacts、review evidence、release evidence 和配置入口。Web Studio 与 Desktop Studio 复用 runtime APIs。

runs settings desktop

Skill

给 Codex、Claude Code、Cursor、Antigravity 或 OpenCode 的协议说明。 Skill 不内置 CLI，入口 agent 通过 PATH 查找 agentmesh。

codex antigravity claude

安装和接入

CLI 用于终端和 Skill 调用；DMG 用于 Desktop Studio；Skill 按 host 单独安装。

1. 安装 CLI

从 Release tarball 安装，或从源码 clone 后 `npm link`。

npm install -g ./agentmesh-0.1.1.tgz
agentmesh doctor --json

2. 安装 Skill

只给实际使用的 host 安装，安装后可用 verify 检查文件状态。

agentmesh skill install --target codex
agentmesh skill install --target antigravity
agentmesh skill verify --target codex --json

3. 打开 Studio

CLI 可启动 Web Studio；DMG 可打开 Desktop Studio。

agentmesh studio --port 4777

# 或安装 AgentMesh.app
DMG-only 只提供 Desktop Studio。 入口 agent 要调用 AgentMesh，仍然需要 PATH-visible 的 `agentmesh` 命令。

支持的工具

Runtime adapter 统一处理命令、prompt、model、reasoning effort、stdout/stderr、 timeout 和 readiness。Antigravity 使用 CLI 内部当前模型。

Adapter Alias 命令 模型语义 适用 stage
codex-cli codex codex exec AgentMesh 传 -m 和 reasoning effort。 plan, execute, verify, review, decide
claude-code-cli claude claude -p AgentMesh 传 --model 和 effort。 plan, execute, verify, review, decide
cursor-agent cursor cursor-agent --print --trust AgentMesh 传 --model，prompt 不能为空。 plan, execute, verify, review, decide
antigravity-cli antigravity agy -p AgentMesh 使用 current，不向 agy 传模型参数。 plan, execute, verify, review, decide
opencode-cli opencode opencode run AgentMesh 传 --model 和 variant。 plan, execute, verify, review, decide
command 无 自定义 executable 无固定模型语义。 按注册能力决定

Workflow 和 packet

Run 创建时会把 workflow、stage assignment、fallback、timeout、context policy 和 release policy 固化到 packet。preset-first UX、decide checkpoint、 current packet schema、fallback/failure policy 和 legacy migration unsupported 是 v0.1.1 的运行语义。

Stage 1 plan 生成计划或候选计划，可 fanout 后 synthesis。
Stage 2 execute 单主控执行代码修改，避免多个 agent 同时写同一批文件。
Stage 3 verify 记录测试、构建、检查命令和验证证据。
Stage 4 review 一个或多个 reviewer 输出 findings 和原始证据。
Stage 5 decide 分类 review 结果，写 decision 或 release verdict。 
agentmesh run \
  --workflow <workflow-id> \
  --plan <planner> \
  --execute <worker> \
  --verify <verifier> \
  --review <reviewer> \
  --decide <decider> \
  --task "实现一个可验证的修改"

Packet 文件

`.agentmesh/runs/<run-id>/` 保存 request、context、status、events、 artifacts、plan、handoff、findings、decision 和 release summary。

status.json events.jsonl artifacts.toml

Reviewer Session：连续审查，但不牺牲独立门禁

flow resume 恢复 AgentMesh Run 的 workflow stage；Reviewer Session resume 在多次 Run 之间恢复 reviewer provider 会话。恢复证据会标记为 non-hermetic， 当前 packet、diff 和验证仍会完整重发。

三种模式

interactive_continuous 用于同一宿主对话的普通连续 review； independent 始终 fresh，release、安全和审批 gate 必须使用； auto 在 scope、adapter 和 registry 不满足时安全退化为 fresh。

当前支持边界

Claude Code 与 OpenCode 当前为 experimental，只验证立即恢复； Codex、Cursor、Antigravity 保持 fresh-only。入口宿主仍可传递安全 scope， 不代表对应 reviewer provider 已启用恢复。

安全和生命周期

空闲 2 小时、绝对最多 12 小时、最多 8 次成功 resume。token 丢失时 fresh， 不得从 workspace、旧 packet 或 provider state 恢复 scope；原始 provider/native ID 不进入 packet、日志或 Studio。 

agentmesh sessions scope create --host codex --json
agentmesh sessions list --json
agentmesh sessions inspect <session-ref> --json
agentmesh sessions close <session-ref> --json
agentmesh sessions close --scope <scope-ref> --json
agentmesh sessions purge --expired --json

连续 review

agentmesh run \
  --workflow <review-workflow-id> \
  --review <reviewer> \
  --decide current \
  --review-session-mode interactive_continuous \
  --host-kind codex \
  --conversation-scope amscope_v1:...

propagated token 只在同一入口对话内原样传递；认证、权限或 trust 失败不会被静默转成 fresh 成功。

配置和安全边界

AgentMesh 是本地文件协议。用户级配置保存本机 agents 和 MCP servers， 项目配置保存 workflow defaults、context policy 和 release policy。

用户配置

~/.config/agentmesh/config.toml 保存 agents、MCP servers、model aliases 和个人偏好。

项目配置

./.agentmesh/config.toml 适合 workflow defaults、default stage agents、fallback 和 context policy。

MCP context

Run 可通过 --mcp-resource 引入 text resource，成功和失败都会写入 provenance。

安全边界

不保存 provider token，不绕过底层 CLI 登录态，Studio API 不提供任意 shell command 路由。

agentmesh agents add --adapter codex --model gpt-5.5 --label "Codex"
agentmesh doctor --json
agentmesh workflows list
agentmesh release-check summary <run-id> --write
Antigravity 模型说明 AgentMesh 不写死 Antigravity 的模型列表。Studio 和 doctor 读取本机可发现的模型信息； 运行时使用 `agy` 当前选择的模型，不向 `agy` 传 `-m`。
