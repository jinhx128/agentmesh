## 20:02 - 固化 GitHub Release 发布流程

- 新增 `scripts/github-release.mjs`，把 Release 资产准备、GitHub Release 创建/补传和远端资产校验收口到同一条可重跑流程。
- 新增 `npm run release:assets`、`npm run release:github`、`npm run release:github:verify`，避免只推送 tag 却没有创建 GitHub Release。
- 新增 `docs/distribution/github-release.md`，明确发布完成定义、标准流程、漏发补救和失败处理；README 打包章节补 GitHub Release 入口。
- 单独 verify 默认校验远端 Release 和资产完整性；创建/补传后强制比较本地 digest，也可用 `--compare-local` 手动比较当前 `dist-release/`。
- 创建/补传 Release 时强制要求 HEAD 等于当前版本 tag 且工作区干净，防止发布后继续改动时误覆盖同版本资产。
